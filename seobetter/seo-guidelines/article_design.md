# SEOBetter Article Design Specification

> **MANDATORY PRE-CHECK: Before making ANY changes to Content_Formatter.php, seobetter.php, or Async_Generator.php, READ THIS FILE and plugin_UX.md first. Verify no features are removed after changes.**
>
> **Purpose:** Defines how generated articles look, render, and perform across any CMS. Every article generated by SEOBetter follows this specification exactly.
>
> **Referenced by:** `Content_Formatter.php` (classic mode), `Async_Generator.php`, `seobetter.php`
>
> **CRITICAL:** Saved drafts use `format_classic()` wrapped in `<!-- wp:html -->`. This outputs a self-contained `<article>` with scoped `<style>` block. ALL styling comes from this scoped CSS. If the draft has no styling, check that `seobetter.php rest_save_draft()` calls `format('classic')` not `format('hybrid')`.
>
> **Last updated:** April 2026

---

## 1. DESIGN PHILOSOPHY

Articles must look like they came from a design-conscious publication (The Verge, Stripe Press, NYT Magazine) — not a Notion page full of emoji headers or a chatbot response.

### Core Principles
1. **Self-contained** — Single `<article>` element with scoped `<style>` block. No external CSS, no JavaScript, no framework dependencies.
2. **Portable** — Works on WordPress, Shopify, Magento, Ghost, or any CMS that accepts HTML.
3. **Scoped** — Every CSS selector prefixed with a unique class (e.g., `.sb-a1b2c3`) to prevent collisions with the host site's CSS.
4. **Responsive** — Fluid typography via `clamp()`, responsive images, mobile-friendly tables.
5. **Performant** — No external requests, lazy-loaded images, minimal DOM nodes.
6. **Accessible** — Semantic HTML, sufficient contrast ratios, no color-only indicators.
7. **Dark mode ready** — `@media (prefers-color-scheme: dark)` scoped to the article wrapper.

## 2. HTML STRUCTURE

```html
<style>
  .sb-[uid] { /* all article styles scoped here */ }
  .sb-[uid] h1, .sb-[uid] h2 { /* heading styles */ }
  /* ... all styles prefixed with .sb-[uid] ... */
  @media (prefers-color-scheme: dark) {
    .sb-[uid] { /* dark mode overrides */ }
  }
</style>
<article class="sb-[uid]">
  <p style="...">Last Updated: Month Year</p>
  <h1>Article Title</h1>
  <h2>Key Takeaways</h2>
  <div class="sb-takeaways">
    <ul><li>...</li></ul>
  </div>
  <!-- content sections -->
  <h2>Section Heading</h2>
  <p>Content paragraph...</p>
  <!-- ... -->
  <h2>References</h2>
  <ol><li><a href="https://real-url.com">Source Title</a> — description</li></ol>
</article>
```

### Rules
- Output a single `<article>` element with class `sb-[6-char-uid]`
- Include one `<style>` tag at the top with every selector prefixed by the scoping class
- Do NOT use Tailwind, Bootstrap, or any framework classes
- Do NOT use `<link>` tags, `@import`, or `<script>` tags
- Return only the HTML block — no markdown fences, no DOCTYPE

## 3. TYPOGRAPHY

### Font Stacks (System Fonts — No External Fonts)
| Context | Font Stack |
|---|---|
| Headings | `ui-serif, Georgia, 'Times New Roman', serif` |
| Body text | `ui-sans-serif, system-ui, -apple-system, sans-serif` |
| Code | `ui-monospace, 'Cascadia Code', 'Fira Code', monospace` |

### Font Sizes (Fluid via clamp)
| Element | Size | Weight | Line Height |
|---|---|---|---|
| H1 | `clamp(1.8em, 4vw, 2.4em)` | 800 | 1.2 |
| H2 | `clamp(1.3em, 3vw, 1.6em)` | 700 | 1.3 |
| H3 | `1.15em` | 600 | 1.4 |
| Body paragraph | `1.05em` | 400 | 1.75 |
| Table header | `0.9em` | 600 | — |
| Table cell | `0.95em` | 400 | — |
| Small/muted | `0.85em` | 400 | — |
| Code block | `0.9em` | 400 | 1.6 |

### Text Wrapping
- Headings: `text-wrap: balance`
- Paragraphs: `text-wrap: pretty`
- Body max-width: `65ch` (optimal reading length)

### Drop Cap
The opening paragraph of the article gets a `::first-letter` drop cap:
- Font size: `3.2em`
- Color: accent color
- Font family: serif (heading stack)
- Float left with small right margin

## 4. COLOR SYSTEM

### CSS Custom Properties (Defined Per Article)
The accent color comes from the user's color picker. All other colors are fixed neutrals.

| Variable | Light Mode | Dark Mode | Use |
|---|---|---|---|
| Accent | User's color (default `#764ba2`) | Same | H2 text, borders, links, table headers, drop cap |
| Text | `#1f2937` | `#e5e7eb` | Body text |
| Heading | `#111827` | `#f3f4f6` | H1, H2, H3 |
| Muted | `#6b7280` | `#9ca3af` | Captions, dates, metadata |
| Border | `#e5e7eb` | `#374151` | Table borders, dividers, HR |
| Surface | `#f9fafb` | `#1f2937` | Blockquote bg, even table rows |
| Background | `#ffffff` | `#111827` | Article background |

### Accent Color Usage
- H2 heading text color
- H2 bottom border (at 13% opacity: `{accent}22`)
- Blockquote left border
- Key Takeaways left border
- Table header background
- Link color
- Drop cap color
- List bullet markers (`::marker`)

## 5. CONTENT BLOCK STYLES

### 5.1 Key Takeaways Box (`.sb-takeaways`)
- Left border: 4px solid accent
- Background: linear gradient (subtle blue-white)
- Padding: 1.25em 1.5em
- Border radius: 0 8px 8px 0 (rounded right side only)

### 5.2 Pros Box (`.sb-pros`)
- Background: light green (`#f0fdf4`)
- Border: 1px solid green (`#bbf7d0`)
- Border radius: 8px
- Used when preceding heading contains "pros", "advantages", "benefits"

### 5.3 Cons Box (`.sb-cons`)
- Background: light red (`#fef2f2`)
- Border: 1px solid red (`#fecaca`)
- Used when preceding heading contains "cons", "disadvantages", "drawbacks"

### 5.4 Ingredients Box (`.sb-ingredients`)
- Background: light amber (`#fffbeb`)
- Border: 1px solid amber (`#fde68a`)
- Used when preceding heading contains "ingredients", "supplies", "what you need"

### 5.5 Callout Boxes (`.sb-callout`)
Detected from paragraph text starting with "Tip:", "Note:", or "Warning:"

| Type | Class | Background | Left Border |
|---|---|---|---|
| Tip | `.sb-callout-tip` | `#eff6ff` (light blue) | `#3b82f6` (blue) |
| Note | `.sb-callout-note` | `#fffbeb` (light amber) | `#f59e0b` (amber) |
| Warning | `.sb-callout-warn` | `#fef2f2` (light red) | `#ef4444` (red) |

### 5.6 Blockquotes
- Left border: 4px solid accent
- Background: surface color
- Border radius: 0 8px 8px 0
- Font style: italic
- Used for expert quotes and pull quotes

### 5.7 Tables
- Full width, border-collapse
- Header: accent background, white text, small caps
- Even rows: surface background (zebra striping)
- Cells: comfortable padding (0.75em 1em)
- Wrapper: `overflow-x: auto` for mobile horizontal scroll
- Subtle box shadow

### 5.8 Code and Pre Blocks
- Inline code: light gray background, small padding, rounded corners
- Code blocks: dark background (#1f2937), light text, rounded corners, horizontal scroll

## 6. ICON RULES (STRICT)

### Inside Generated Articles
- **NEVER** place icons next to headings, list items, or inline with sentences
- **NEVER** use emoji as icons in body copy
- No checkmark bullets, no decorative icons beside H2s, no emoji headers
- This is the single clearest tell that content was AI-generated

### Icons Permitted ONLY In
1. Small SVG glyphs inside callout box corners (info icon, sized 1em, `currentColor`)
2. A single icon in the Key Takeaways box header (not per item)
3. Social share or author byline sections

### When Icons Are Used
- Embed as inline SVG directly in the HTML
- Do NOT reference Lucide or Iconify as runtime dependencies
- Copy raw SVG path data and inline it
- Size: no larger than 1em
- Color: `currentColor` (inherits text color)

### Default Behavior
Generate articles with **zero icons** in the body. Only include if template explicitly calls for a callout or byline component.

## 7. IMAGE REQUIREMENTS

```html
<figure>
  <img src="URL" alt="Descriptive alt text"
       loading="lazy" decoding="async"
       style="aspect-ratio:16/9;object-fit:cover" />
</figure>
```

| Attribute | Value | Purpose |
|---|---|---|
| `loading` | `lazy` | Defer off-screen images |
| `decoding` | `async` | Non-blocking decode |
| `aspect-ratio` | `16:9` (inline style) | Prevent CLS |
| `max-width` | `100%` (via scoped CSS) | Responsive |
| `border-radius` | `8px` (via scoped CSS) | Visual consistency |

## 9. PERFORMANCE

### Loading
- No external CSS files or @import
- No JavaScript in article output
- Lazy-loaded images (`loading="lazy"`)
- `decoding="async"` for non-blocking image decode
- `aspect-ratio` prevents cumulative layout shift

### Render
- Scoped CSS limits style recalculation to article element only
- No CSS animations by default (saves paint cycles)
- Minimal DOM nodes — semantic HTML without wrapper divs

### Indexing
- Semantic HTML (article, h1-h3, p, ul, ol, table, blockquote, figure)
- Schema JSON-LD in `<head>` via wp_head hook (separate from article HTML)
- All text content in proper HTML elements (not hidden in attributes or data-)
- Links are real `<a href>` tags crawlable by search engines and AI agents

---

## 10. REFERENCES SECTION

Every article MUST end with a References section containing:
- Real URLs from DuckDuckGo search + category APIs + country APIs
- Clickable markdown links rendered as `<a href="https://real-url.com">Source Title</a>`
- Each reference: numbered, with source title linked and one-sentence description
- No hallucinated URLs — only cite sources from research data
- 5-10 references minimum

```html
<h2>References</h2>
<ol>
  <li><a href="https://thekitchn.com/sourdough-guide">The Kitchn</a> — Comprehensive beginner's guide to sourdough bread making.</li>
  <li><a href="https://www.kingarthurbaking.com/recipes/sourdough">King Arthur Baking</a> — Detailed sourdough recipes with troubleshooting tips.</li>
</ol>
```

## 11. CONTENT TYPE VARIATIONS

Different content types may adjust the design:

| Content Type | Design Adjustments |
|---|---|
| Recipe | Ingredients box (amber), numbered step styling, prep/cook time metadata |
| Review | Pros/cons boxes (green/red), verdict section, rating display |
| Comparison | Mandatory comparison table, criterion winner indicators |
| News Article | Short paragraphs, no drop cap, dateline styling |
| FAQ Page | Q&A card styling for each pair |
| How-To | Numbered step cards, prerequisite box |

## 12. DESIGN CHECKLIST

Every generated article MUST have:
- [ ] Scoped `<style>` block with unique class prefix
- [ ] `<article>` wrapper with matching class
- [ ] System font stacks (no external fonts)
- [ ] Fluid heading sizes via `clamp()`
- [ ] Drop cap on opening paragraph
- [ ] Dark mode support via `@media (prefers-color-scheme: dark)`
- [ ] Accent color applied to: H2, borders, links, table headers, drop cap
- [ ] Responsive tables with `overflow-x:auto`
- [ ] Lazy-loaded images with `aspect-ratio`
- [ ] Context-aware styled blocks (takeaways, pros/cons, callouts)
- [ ] No emoji in article body
- [ ] No inline styles on individual elements (CSS classes instead)
- [ ] No external dependencies (CSS, JS, fonts)
- [ ] References section with real clickable URLs
- [ ] Semantic HTML throughout (article, headings, p, ul, ol, table, figure)
