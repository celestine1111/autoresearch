# SEOBetter — Twitter Growth Playbook

> AI-driven Twitter / X strategy for SEOBetter, a WordPress plugin that generates content optimized for AI search engines (ChatGPT, Perplexity, Gemini, Google AI Overviews, Claude).
>
> **Primary goal:** Grow from 0 → 5,000 followers in 12 months and convert ≥1% to Pro subscribers ($39-179/mo) — translating to 50+ paying users from organic Twitter alone.
>
> **Secondary goal:** Build founder credibility in the AI-SEO / GEO niche so the brand becomes the obvious answer when WordPress site owners ask "how do I optimize for ChatGPT?"
>
> **Last updated:** 2026-05-02

---

## 1. Why Twitter — and why now

The AI search shift (~30% of search volume now happens inside ChatGPT / Perplexity / Gemini / Google AI Overviews vs. blue links — Princeton 2024, KDD 2024) is the wedge. Twitter / X is where:

- WordPress builders, indie hackers, SEOs, and content marketers congregate
- The conversation about "is SEO dead?" is happening daily
- AppSumo deal hunters and small-agency owners discover tools
- Build-in-public works — Mike Sutton (RankMath), Joost (Yoast), Cyrus Shepard, Aleyda Solis are all there
- Founders with under 10K followers have shipped products to 100K+ buyers via consistent posting

The category is wide open: there's no single "the AI-SEO / GEO person on Twitter" yet. SEOBetter can claim that territory if we move fast.

## 10. Account safety / sustainability

**Don't get suspended.**

- Twitter's automated-account policy: agent-driven posting is fine if it's labeled in bio as such ("AI agent powered by SEOBetter, run by @benhandle"). Quietly automating without disclosure violates the developer agreement.
- Engage authentically. Don't mass-follow or mass-DM.
- Hard ceiling: 50 replies / day, 10 follows / day, 0 mass-DMs. The agent should respect these limits.
- Don't use shortened URLs (bit.ly, t.co) outside the auto-shorten Twitter does itself — they correlate with spam classification.

**Don't burn the brand.**
- Every tweet is reviewed by a human (Ben) for the first 90 days before posting. Build a queue, approve in batches.
- After 90 days, sample 10% of posts for review. Quality drift kills accounts faster than slow growth.
- Hard rule: never reply with sales pitches in someone's emotional thread (loss / illness / personal news). The agent must detect emotional content and skip.

**Don't argue.**
- If someone trash-talks the plugin, the agent ignores. Founder (Ben) handles publicly only when the criticism is technical and addressable. Personal attacks → block + move on.

## 14. Common mistakes to avoid (curated from indie SaaS Twitter postmortems)

| Mistake | Why it kills accounts | What to do instead |
|---|---|---|
| Posting only product features | Reads as sales-y, kills algorithmic reach | 60% insights, 25% product, 15% other |
| Generic "growth hacker" tone | The market is saturated and skeptical | Specific data > universal advice |
| Inconsistent posting | Algorithm de-ranks intermittent accounts | Daily minimum, even if short |
| Replying only to celebrities | Big accounts ignore replies; mid-tier converts | Tier A + Tier B + Tier C mix |
| Buying followers | Twitter detects → shadowbans within weeks | Slow organic growth |
| AI-detectable copy | Hurts brand credibility ("you wrote that with ChatGPT") | Run the humanizer Claude skill on every tweet |
| Linking everywhere | Twitter de-ranks links in single tweets | Bio link only; threads can self-link last tweet |
| Long threads daily | Audience burnout | One thread per week max |
| Engagement bait ("Repost if you agree") | Twitter actively penalizes this | Earn engagement with substance |
| Ignoring DMs | Conversion happens in DMs, not in public | Set up agent autoresponder + human review |

## 16. Crisis playbook

**Scenario: Plugin has a public bug / outage**
- First response (within 2 hours): acknowledgement tweet, no excuses, ETA on fix
- Followup (within 24 hours): root cause + what we changed
- Wrap (within 48 hours): one-tweet retrospective with lesson learned

**Scenario: Competitor mentions us negatively**
- Default: ignore. The mentioned-by-competitor halo effect outweighs the criticism.
- Exception: if criticism is factually wrong, founder (not agent) replies once, calmly, with proof. Block thread reply expansion if it gets ugly.

**Scenario: User accuses us of plagiarizing / scraping**
- Response: link to `external-links-policy.md` (publicly published) showing we use a research pool with verified URLs and the AI is bounded to citing only those. Show a screenshot of the validator output.

**Scenario: The agent posts something embarrassing**
- Founder deletes immediately
- Acknowledge if it gets traction ("Our AI agent posted something off-brand earlier. Removed. Improving the system prompt.")
- Update the banned-pattern list and the system prompt
- Pause auto-posting for 48 hours, full human review
